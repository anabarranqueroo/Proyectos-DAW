Manual de Configuración de WildFly

· WildFly ·

¿Qué es WildFly?

WildFly es un servidor de aplicaciones Java EE / Jakarta EE de código abierto desarrollado por Red Hat. Es la evolución de JBoss AS, ofreciendo una arquitectura modular y ligera, ideal para ejecutar aplicaciones empresariales Java modernas. Es conocido por su alto rendimiento, flexibilidad y soporte de las últimas especificaciones Jakarta EE.

COMPONENTES

- Undertow. Servidor web y contenedor de servlets ligero y de alto rendimiento.
- JBoss Modules. Sistema modular que permite cargar servicios y librerías bajo demanda.
- Infinispan. Sistema de caché distribuido de alto rendimiento.
- Remoting y EJB. Permiten la comunicación remota y gestión de Enterprise JavaBeans.
- CLI Management. Consola de administración por línea de comandos.
- Web Admin Console. Interfaz gráfica de administración vía navegador.

VENTAJAS

- Alto rendimiento y escalabilidad.
- Soporte completo para Jakarta EE 10.
- Administración flexible mediante consola web o CLI.
- Arquitectura modular (solo carga lo necesario).
- Compatible con entornos Windows, Linux y macOS.
- Integración con Maven, Eclipse y VS Code.

Requisitos previos para su instalación

JDK 11 o superior.

Tener instalado JDK 11 o una versión más reciente. Puedes comprobarlo ejecutando en la consola: javac -version.

Variables de entorno configuradas.

Debe estar configurada la variable JAVA_HOME apuntando a la ruta del JDK y añadida en PATH con %JAVA_HOME%\bin.

Sistemas Operativos compatibles.

Windows 10, Windows 11, Linux, macOS.

Espacio mínimo en disco.

300 MB (recomendado 500 MB).

RAM mínima.

1 GB (recomendado 2 GB o más).

¿Cómo instalamos WildFly en Windows?

Para instalar WildFly, vamos a su página oficial de descargas: https://www.wildfly.org/downloads/, y descargamos la versión más reciente (archivo ZIP).

1. Una vez descargado, descomprimimos el archivo en la carpeta donde queremos instalar WildFly.
2. Dentro de la carpeta veremos varias subcarpetas como bin, standalone y domain. WildFly viene con dos modos de ejecución:
- Standalone mode: Configuración simple, ideal para entornos de desarrollo.
- Domain mode: Permite gestionar múltiples instancias o servidores WildFly al mismo tiempo.
3. Para iniciar el servidor en modo standalone, abrimos una terminal en la carpeta bin y escribimos: 
                        
                            ./standalone.bat
En unos segundos, WildFly arrancará escuchando por defecto en el puerto 8080. Podemos comprobarlo abriendo: http://localhost:8080.
4. Para detener el servidor, presionamos Ctrl + C en la terminal o ejecutamos: 
                        
                            jboss-cli.bat --connect command=:shutdown

¿Cómo se configura WildFly?

1. Para acceder al panel de administración de WildFly, primero debemos crear un usuario administrativo. En la carpeta bin, ejecutamos: 
                        
                            add-user.bat
2. El asistente nos preguntará:
- Tipo de usuario: seleccionamos Management User.
- Nombre de usuario y contraseña.
- Confirmamos las opciones y finalizamos.

3. Una vez creado el usuario, accedemos al panel de administración desde: http://localhost:9990 e iniciamos sesión con las credenciales creadas.
4. Desde el panel podemos gestionar despliegues, configuraciones, módulos y recursos del servidor.
5. Si queremos cambiar el puerto HTTP o HTTPS, modificamos el archivo standalone.xml ubicado en standalone/configuration: 
                        
                            <socket-binding name="http" port="8080"/>
                            <socket-binding name="https" port="8443"/>
Cambiamos los valores de port según nuestras necesidades y guardamos el archivo. Luego reiniciamos el servidor.

¿Cómo desplegamos una aplicación web en WildFly?

1. Podemos desplegar aplicaciones .war o .ear en WildFly de dos formas:
Desde la consola web

Accedemos al panel de administración (http://localhost:9990), seleccionamos Deployments → Add → Upload Deployment y elegimos el archivo .war.

Desde la línea de comandos

En la carpeta bin, ejecutamos el CLI con:

                        
                            jboss-cli.bat --connect deploy C:\ruta\miAplicacion.war

Para detener o eliminar el despliegue:

                        
                            undeploy miAplicacion.war
2. Luego accedemos a la aplicación desde el navegador, por ejemplo: http://localhost:8080/miAplicacion.
