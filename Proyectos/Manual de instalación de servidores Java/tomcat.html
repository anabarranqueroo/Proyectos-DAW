Manual de Configuración de Apache Tomcat

· Apache Tomcat ·

¿Qué es Apache Tomcat?

Apache Tomcat es un servidor de aplicaciones de código abierto que implementa especificaciones de servlets y JSP. Proporciona un entorno confiable y eficiente para ejecutar aplicaciones web Java, permitiendo procesar solicitudes dinámicas y entregar contenido web generado por servlets y JSP. Tomcat actúa como intermediario entre los clientes y las aplicaciones Java, ofreciendo un entorno seguro y optimizado para su ejecución.

COMPONENTES

- Catalina. Contenedor de servlets responsable de ejecutar las aplicaciones Java.
- Coyote. Conector HTTP que administra las solicitudes web entrantes.
- Jasper. Motor que compila y ejecuta páginas JSP convirtiéndolas en servlets.

VENTAJAS

- Es gratuito y de código abierto.
- Instalación y configuración sencillas.
- Seguridad configurable e integrada.
- Alta estabilidad y confiabilidad en producción.
- Escalable y personalizable según los requerimientos del proyecto.
- Amplia comunidad de soporte y documentación.

VERSIONES DISPONIBLES:
  • Tomcat 9. Compatible con Java EE 8 / Servlet 4.0
  • Tomcat 10. Compatible con Jakarta EE 9 / Servlet 5.0
  • Tomcat 11. Compatible con Jakarta EE 10 / Servlet 6.0

Requisitos previos para su instalación

JDK 8 o superior.

Se debe contar con Java Development Kit (JDK) versión 8 o superior. Verifique la instalación ejecutando en la consola: javac -version.

Variables de entorno configuradas.

Es necesario que las variables de entorno estén correctamente definidas: JAVA_HOME con la ruta del JDK, CATALINA_HOME con la ruta de instalación de Tomcat y PATH incluyendo %JAVA_HOME%\bin.

Sistemas Operativos compatibles.

Windows 10 (64 bits), Windows 11 (64 bits), Windows Server 2016, 2019 y 2022.

Espacio mínimo en el disco.

Se requiere un mínimo de 200 MB de espacio libre en disco para la instalación.

RAM mínima.

Se recomienda disponer al menos de 512 MB de memoria RAM para un funcionamiento óptimo.

¿Cómo instalamos Apache Tomcat en Windows?

Para instalar Apache Tomcat, acceda a la sección de versiones disponibles y seleccione la versión deseada o ingrese directamente a la página oficial: https://tomcat.apache.org/.

En el sitio oficial, seleccione la versión correspondiente y haga clic en el instalador 32-bit/64-bit Windows Service Installer.

1. Ejecute el instalador, acepte la licencia y siga las instrucciones del asistente de instalación.
2. Seleccione los componentes que desea instalar según las necesidades del entorno.
3. Configure el puerto de servicio, así como el usuario y contraseña en el panel de configuración básica.
4. Indique la ruta de la máquina virtual de Java y la carpeta de instalación de Tomcat.
5. Para verificar la instalación, abra en un navegador: http://localhost:8080.

¿Cómo se configura Apache Tomcat?

Para agregar o modificar usuarios, edite el archivo tomcat-users.xml ubicado en la carpeta conf de Tomcat.

1. Antes de la etiqueta de cierre </tomcat-users>, agregue las siguientes líneas de configuración: 

                        <role rolename="manager-gui"/>
                        <role rolename="admin-gui"/>
                        <user username="nombre_usuario" password="contraseña_usuario" roles="manager-gui,admin-gui"/>
2. Guarde los cambios y reinicie Tomcat para que las modificaciones surtan efecto.

¿Cómo desplegamos una aplicación web en Apache Tomcat?

Localice el archivo .war generado por su proyecto en la carpeta dist y cópielo en la carpeta webapps de Tomcat. Posteriormente, acceda mediante navegador a https://localhost:8080/nombre_del_proyecto para verificar su correcto despliegue.
